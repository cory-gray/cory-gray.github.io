Lexmark Café Customer Forecasting

Spring 2016

Key Concepts: Big Data, Machine Learning, Forecasting, Web Applications
Languages and Tools: R, Shiny, Java, Excel

As part of a senior capstone project, a group consisting of myself and two classmates created a web application which predicts customer activity at a café on the campus of Lexmark International, Inc. The product is designed to aid the café owner or manager to accurately estimate staffing and supply needs over a user-modifiable time scale with different parameters, allowing more efficient utilization of resources and ultimately increasing net profits.

We began with a large data set collected from the cafe grounds, which was parsed and analyzed using Java, C++, and Microsoft Excel. We then used data mining and machine learning techniques in conjunction with R to obtain predictive models, which were implemented into a web application using Shiny.

I served primarily as the project manager, establishing deadlines for project milestones, breaking up and assigning tasks to group members, and regularly communicating with the client to establish program specifications and relay progress updates. I was also responsible for most of the project documentation, available at the project website in the link below.

Solar Ace

November 2015

Key Concepts: Computer Graphics, Physics, Computer Games
Languages and Tools: Java, OpenGL (JOGL)

As the final project in a course in graphics and multimedia, I and a partner created a space-themed racing game, wherein the player must race against the clock through floating rings scattered about an alien star system, whilst dodging or destroying the asteroids which spawn and fly towards the player’s ship. While my partner laid the framework for movement and passing checkpoints, I implemented the game obstacles (planets and asteroids), collision detection, the asteroid spawning “AI”, and the shooting mechanism. The game was written in Java and uses JOGL, a Java binding for OpenGL, for rendering graphics.

Voidhack

Fall 2013

Key Concepts: Computer Games, Procedural Generation
Languages and Tools: C++, ncurses, XML

One of my first significant software projects as well as one of my favorites, Voidhack is a game heavily based on Rogue, and is the result of a semester-long course project. In Voidhack, the player must fight through 50 procedurally generated levels of an overrun space station, facing more powerful foes as they progress. The game currently features 131 items, including weapons, armor, and health items, as well as 75 different enemies. It was written in C++ and uses ncurses to display the game world and XML to store game data.
